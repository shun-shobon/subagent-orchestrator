# マージ・統合ガイド

## 統合戦略

- 依存関係が満たされたタスクのみ統合する。
- 依存レイヤーごとに段階的に統合する。
- 統合コミットは追跡しやすく保つ。

## 統合チェックリスト

1. レビュー承認済みであることを確認する。
2. DoD検証が通過していることを確認する。
3. 必要ならベースブランチを取り込む。
4. 統合ブランチで `git merge --no-ff <task-branch>` を実行して取り込む。
5. スモークテストと対象回帰テストを実行する。
6. 結果を `orchestration/integration-log.md` に記録する。

## 競合対応

- 競合は統合ブランチ上で解決する。
- 解決方針と理由を記録する。
- 挙動に影響する解決をした場合はレビュー再確認を依頼する。

## 完了条件

- すべてのタスクが `done`。
- 未解消のブロッカー指摘がない。
- 引継ぎ文書が最新で完結している。
